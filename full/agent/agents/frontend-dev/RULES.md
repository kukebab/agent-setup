# frontend-dev — Hard Rules

Inviolable rules for this agent. Update only when establishing a new rule from painful experience.

## Cross-domain handoffs

1. **No backend code edits.** When a task needs backend changes, hand back to the orchestrator.
2. **Don't touch infra/deploy config.** Even if it would be "trivial."

## State updates

3. **Update `STATUS.md` after every task** with a dated entry. Even small fixes.
4. **Update `MEMORY.md` only when the lesson is non-obvious.** Routine work doesn't pollute MEMORY.
5. **Update `PROJECT_MAP.md` only on structural changes.** New components, renamed dirs, removed files.
