# PDF Export — MVP Spec

**Date:** 2026-05-03
**Owner:** Alex (backend)
**Driver:** Maya (founder)
**Customer:** [[clients/bluefin-coffee]]
**ETA:** 2026-05-08 (Friday)

## Problem

Bluefin Coffee uses Acme Notes for daily SOPs. They print these for shift handovers. Currently they screenshot the editor view, which loses formatting and looks unbranded.

## Goal

Let any user export a single note as a branded PDF in 1 click.

## Out of scope (v1)

- Multi-note exports (just one at a time)
- Tables, embedded images, advanced formatting
- Custom templates per user/team
- Scheduled/recurring exports

## Acceptance criteria

Given a user is viewing a note, when they click "Export to PDF", then:

- A PDF downloads within 3 seconds
- The note's body renders as plain text + headings (Markdown rendered)
- The PDF has a header with the team's logo + name
- The PDF has a footer with date generated and "Generated by Acme Notes"
- File name: `<note-title>-<YYYY-MM-DD>.pdf`

## Technical approach

- Use `puppeteer` to render Markdown → HTML → PDF (battle-tested)
- Markdown rendering library: `marked` (already in deps)
- Branding: pull team's logo + name from their settings
- Cache rendered PDFs for 5 min (idempotent re-downloads)

## Risks

- Puppeteer adds ~150MB to backend container — ok for now, evaluate alternatives if it bites
- High-frequency exports could overload single backend node — add rate limit (10/min/user)

## Rollout

- Ship to Bluefin first (test with their real SOPs)
- 48h soak
- Roll out to all design partners
- Public at launch
